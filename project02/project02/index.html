how do you write a one-line comment Two forward slashes prior to the comment you want to type will comment out the rest of the line. // Comment here.
how do you write multi-line comments Multiple line comments begin with a forward slash and asterisk, then the comment, then asterisk and forward slash close the multiple line comment. /* Comment here in multiple lines */
how do you declare a variable A variable that will change later in the application is declared with "let" followed by the variable identifier.
A variable that does not change during the application can be declare with "const" followed by the variable identifier.		 let changableVarName;
const unchangingVarName
how do you assign a textual value to a variable Textual values can be assigned using the assignment operator, =, followed by either opening and closing single quotes (aka apostrophes) or opening and closing double quotes, where the value to be assigned sits in between the pair of quotes. let name = "Libby";
let name = 'Libby';
when do you use 'apostrophes' vs. "quotation marks" Either one can be used, as long as you're consistent. let varName = "This is okay to use";
let var2Name = 'This is okay too';
when do you use the backwards slash \ The backwards slash can be used to escape special characters. let varName = "This is okay to \n use";

Review Questions:

The JavaScript in this project runs after the rest of the body of the HTML code is rendered in the browser.

The + is used for concatenation. This is an available method to use for both strings and numbers. However, JavaScript will automatically convert the number to a string if these two different primitive data types are attempted to be concatenated. When multiplication is used between a string and number datatype, there is not a way for JavaScript to handle that, so NaN, not-a-number, is output as the value.

TextContent is part of the Document Object Model. It uses straight text and is faster then the innerHTML option. InnerHTML parses the string passed as HTML, so it takes a step longer for the browser to render the result.

JavaScript is built to automatically insert a semicolon to the end of a JavaScript statement automatically. This is very different from other programming languages. It is not necessary, but makes for easier reading if a semicolon is typed at the end of each statement.

It is okay to break a statement onto two lines in JavaScript when a statement is very long. In this situation, break the line after an operator, and no special character is necessary. To do it within a text string, close the string with it's matching closing quote and use the + operator to join the enclosed string on the next line.
