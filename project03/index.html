Function Terminology

term definition
function A function allows the developer to group statements together that perform a specified task.
statement In JavaScript, a statement is an individual step or instruction that ends with a semicolon.
parameter A parameter is a piece of information provided to a function in order for the function to complete its specific task. They represent the values that will be passed to the function when the function is called.
argument An argument refers to the specific values that are used to calculate the return value for a function.
return value This is the expected response from a function after a task is preformed.
function declaration A function declaration creates a function containing the code used to perform that function's task when it is called in the code.
function expression A function expression is placed where one might expect to see an expression. It may or may not have a name and is located where the interpreter would normally see an expression.
anonymous function An anonymous function is a function without a name.
immediately invoked function expression These functions do not have names. They are executed once the interpreter encounters them.

A function without parameters can be called with empty parenthesis following the variable name, then ending with a semicolon: showTotal();.

The code, showTotal(2,10); can be used to pass the 2 for the tax and 10 for the total parameters.

Variable Scope Questions

The "use strict" statement at the beginning of the javascript file forces the developer to explicitly declare all variables.

There are two ways to create a global variable. The first is declaring the variable in the javascript file that is not inside of a function. This will declare the variable at the global, or window-level. The second way is to not use the keyword var prior to the variable name.

The keyword used to create a block-level variable is var. The let and const keywords are also used to declare variables.

Object Terminology

term definition
object Objects are a model of something in the real world. They contain a set of variables and/or functions.
property Properties are variables that are a part of an object.
method Methods are functions that are a part of an object.
key A key is the name of a property or method of an object.
literal notation Literal notation is explicitly stating each key and value of an object between curly braces, after declaring the object's name. Each key and its value are separated by a colon. Each property and method are separated with a comma, exclude the comma after the last property or method.
dot notation Dot notation allows access to the properties or methods of an object by placing a period (.) after the object name, followed by the property or method that needs to be accessed.
constructor notation Using the new Object(); notation after the declared object name allows the developer to create a blank object that can then be appended with the specific key/values for it.
this The keyword this is used inside of functions and objects to refer to the specific instance of the object that is calling it. If used outside of a scope block, it refers to the window object.

User-Created Object Questions

Literal notation allows you to individually assign each property value within it according to what the developer requires. This is ideal if you're just needing one specific object with values or methods that are specific to it.

Constructor notation is highly efficient if you need to make a lot of instances of a single object. The constructor notation creates a sort of generic template for you to assign values as needed to create the object you need. This means you can tweak the objects differently, while making more than a few of them.

If a function is created in the global scope, it is built where the window function is the highest and default object. When the keyword this is used within the function, it refers to the window object and will respond as such.

When the function created is within an object, the keyword this refers to the object that is calling the function. It would NOT refer to the window object.
