HTML - CSS - JavaScrpt Essential Terminology

Terminology Language Purpose Example
element HTML to semantically describe and structure content on a web page <p>These tags and text constitute a p (paragraph) element.</p>
attribute HTML Attributes tell more about the element with which they are associated. In the following code, the "xyz" is the attribute: <p id="xyz">
selector CSS Indicates which html element or elements that a specific CSS rule apply to. In the following code, the selector shown is for .holidayItem: .holidayItem {color: pink;}
property CSS Property refers to an aspect of the selector that you want to control or modify. In the following code, the property name is color and the property vale is pink: .holidayItem {color: pink;}
declaration CSS The declaration refers to the block of code that comes after the selector. This includes the curly braces which surround the property name(s) and value(s). In the following code, the declaration block comes after .fruit: .fruit {color: pink;}
object JavaScript An object is a collection or representation of associated data In the following code, today is considered an object: var today = new Date();
method JavaScript A method represents things that can be done with an object. In the following code, getHours() is a method: today.getHours();
event JavaScript An event is a point of interaction with a user, another computer, or another object. An example would be when a user clicks on a button while on a webpage.
property JavaScript A property is a characteristic of an object that has both a name and a value. This can be any number of characteristics, including the color of the object, what type of object it is or what it's current state is (active, static, paused)

How do HTML id and class attributes connect to CSS selectors?

  1. <p id="xyz">Item number xyz.</p> connects to what selector in the css? #xyz {}
  2. How many html elements may have the same id="xyz" attribute value on a given page? Only one, unique id value may appear on one page.
  3. <p class="holidayItem">Seasonal item for sale.</p> connects to what selector in the css? .holidayItem {}
  4. How many html elements have the same class="holidayItem" attribute value on a given page? There can be as few or as many as the developer requires.

Essential JavaScript best practices

  1. inline vs. embedded vs. external JavaScript using examples.

    Pro: It is considered best practice to code an external JavaScript file that is externally linked to the web-page. This allows for re-usability, maintainability and clear organizational separation. It allows the developer to delay the execution of events until a certain thing occurs.

    Con: Inline JavaScript is executed immediate when the page is rendered. This doesn't allows for user interaction if everything must be executed at once.

    Code Example: Inline JavaScript is written between script tage: <script>put JavaScript here<script> This is different from an externally referenced javascript file, which is included right before the closing html body tag as follows: <script src="javaScriptTemplate.js"></script>

  2. indenting your code.

    Pro: Creates very easy to follow, clean and organized code. This is key for ease of use and maintainablity, especially if you're part of a team.

    Con: Can be time consuming at first.

    Code Example: Un-indented code...<html><body>Content</body></html>

  3. inserting code comments.

    Pro: Gives the code some context, especially if there are many sections that are doing different tasks. This is incredibly invaluable in a team setting, as well as for yourself.

    Con: Takes time. Can get excessive if done without a critical eye.

    Code Example: Use <!-- Comment here --> anywhere in the code that will help someone without any prior knowledge of the project for which the code was written.

Java vs JavaScript

  • What is the general purpose for each language?

    • Java is a complied programming languages that can be used to build applications or for more backend processes

    JavaScript is a scripting language that started as a way to interact with users of web pages. Since then, it has become useful in a range of ways, from making online browser games, web page based applications and even server side scripting.

  • Strict vs loose variable typing - what does this mean as it relates to Java and JavaScript?

  • Java is considered a strict language. This means that when variable are declared, the type of the variable must be explicitly stated. This variable type will not change for the entirety of the program, although the value of it may.

    JavaScript is a loose variable typed language. This means that a variable type need not be declared, as JavaScript can keep track of all that for you. It also allows variables to change types as the program is written or interpreted.

  • Where is the source code for each language complied or interpreted?

    • Java is complied through a compiler. Often this is built in to the IDE a programmer is using. For instance, Visual Studio. Whereas, JavaScript is interpreted by the interpreter within the browser of the vistor.
